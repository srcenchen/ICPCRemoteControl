# ICPC 无锡学院 Linux 集控系统 — 需求与实现文档

## 技术选型
*   **后端开发语言**: Go (Go 1.26+)
*   **数据库存储**: SQLite3 (纯 Go 版本，无 CGO 依赖)
*   **前端框架/库**: jQuery, CodeMirror (代码编辑器), xterm.js (网页终端), Bootstrap-like 响应式后台样式，所有前端静态资源直接使用 Go 编译内嵌到 Server 二进制程序中。
*   **自定义网络传输协议**: 依靠底层的自研 `go-silver-core` (GSP 协议) 进行 TCP 消息流的高效、可靠通信。

## 功能需求与系统实现细节

### 1. 服务发现与网络连接 (Avahi mDNS)
*   **服务端发布**: 服务端启动时，通过 `avahi-publish -a -R icpc-server.local 0.0.0.0` (或使用内建自动绑定机制) 在局域网发布本地 IP 的 mDNS 地址。
*   **客户端发现**: 
    1.  客户端默认通过 `icpc-server.local` 解析服务端。
    2.  **冗余机制**: 客户端启动时会在用户目录自动创建 `server` 空文件。若此文件不为空，则优先读取此文件内容 (一个 IP 或域名) 作为服务端地址。
*   **可靠长连接**: 客户端与服务端通过定制的 TCP 长连接进行双向异步通信。
    *   客户端以 **15秒/次** 的频率向服务端发送心跳 `ping` 消息。
    *   服务端开启 TCP Keep-Alive 并设置探测间隔为 **5秒**。
    *   如果在 **30秒** 内没有收到任何数据包 (或者客户端断开)，服务端或客户端将判定连接失效并断开旧套接字，客户端随即启动断线重连机制 (默认重试间隔 **5秒**)。

### 2. 客户端 ID 自动分配与主机名自动更改
*   **唯一性分配**: 选手机连接后，服务端会在 SQLite3 中原子化分配一个全局递增的唯一整数 ID (从 1 开始)。
*   **ID 固化**: 客户端获取 ID 后，会固化写入本地文件 `/var/lib/icpc-client/id`。即便客户端离线、进程重启或系统重启，依然会在重连时请求原有的 ID。
*   **主机名修改**: 客户端程序运行在 **root** 权限下，接收到服务端分配 of ID 后，自动执行 `hostnamectl set-hostname {prefix}-{ID}` 和 `hostname {prefix}-{ID}` (默认前缀为 `cwxu-icpc`) 修改主机名。

### 3. 设备硬件与系统信息收集 (Fastfetch JSON)
*   客户端连接成功后，在后台调用 `fastfetch --format json` 获取本机的软硬件信息。
*   客户端将输出的 JSON 数组结构化发送给服务端。服务端解析并录入数据库，在设备表中实时维护以下信息：
    *   **基础信息**: ID、主机名、登录用户名、MAC地址、IP地址、上次上线时间、首次上线时间。
    *   **操作系统**: OS Pretty Name、架构、内核版本。
    *   **核心配置**: CPU 型号/物理核/逻辑核、GPU 型号/厂商、总内存与空闲内存大小、硬盘总空间与挂载点。
    *   **桌面环境**: 终端类型 (Konsole/Bash)、当前桌面环境 (DE) / 窗口管理器 (WM) / WM主题 / 字体等。
    *   **软件包计数**: dpkg/pacman/flatpak/nix 等不同包管理器的已安装软件数量。

### 4. 远程批量命令执行
*   **下发形式**: 控制台后台可以对单台、多台或全部在线的选手机下发 shell 脚本或直接运行命令。
*   **执行保障**: 客户端在收到命令请求后，通过 `sh -c` 执行命令，设置命令运行组（Setpgid），便于统一管控生命周期。
*   **实时流式输出**: 执行过程中的标准输出 (stdout) 和标准错误 (stderr) 会以行为单位实时通过 TCP 连接推送回服务端，并在 SQLite3 中持久化。
*   **超时控制与取消**: 命令设置默认执行超时时间 (如60秒)，同时支持管理员在控制台手动强行取消正在执行的命令 (客户端会对目标进程组下发 `SIGKILL` 强杀信号)。

### 5. Web Terminal 交互终端
*   **底层集成**: 客户端结合 Go 语言的 `pty` 库，在接收到 `terminal_open` 消息后，会在选手机上以 root 身份开启伪终端交互式 bash 会话。
*   **实时控制**: 服务端通过双向 WebSocket 消息流中转，支持将键盘输入、大小缩放 (xterm resize)、关闭通知实时同步到 client shell 中，前端采用高性能的 `xterm.js` 图形界面呈现。

### 6. 考场签到与签退系统
*   **选手机本地签到页面**: 客户端在本地 `8090` 端口开启微型 Web 服务，自动渲染包含姓名、学号等表单的 `checkin_page.html` 签到控制面。
*   **签到流程**:
    1.  选手使用本机浏览器打开 `http://localhost:8090` 访问。
    2.  填写“姓名”和“学号”提交，客户端捕获请求并将其转派给 TCP 长连接以向服务端申请确认。
    3.  服务端记录签到时间、学生基本信息和签到状态 (已签到、已签退) 并返回响应。
*   **自定义签到文案与签退后置命令**:
    *   管理员可在服务端设置全局签到欢迎词、警告注意事项、签到成功后的祝词、签退后的说明语。
    *   **后置脚本**: 支持配置签退后自动执行的脚本指令 (例如 `shutdown -h now` 或清理桌面的 Shell 命令)，让选手完成比赛退场时机器自动安全关机。

### 7. 桌面半透明水印防作弊机制
*   **底层渲染**: 客户端内置 X11 水印覆盖机制，通过 `github.com/jezek/xgb` X Go 绑定与 X11 的 Shape 形状扩展库建立图形窗口。
*   **无感知防作弊特性**:
    *   自动在 root 用户的 GUI 进程列表中逆向抓取普通用户的 `DISPLAY` 与 `XAUTHORITY` 权限凭证，确保能成功挂载至 X11 显示环境 (`:0`)。
    *   绘制一个浮动在最上层的半透明文本信息水印，包含：**选手设备ID、物理 IP 详细信息、当前选手的签到状态与学号姓名**。
    *   利用 X11 Shape Extension 建立的非标准透明层，该窗口对选手的鼠标点击、键盘录入完全穿透，不影响选手的常规打题与 IDE 使用，起到纯粹的背景/浮动水印展示和防代考防作弊效果。

### 8. 实时屏幕监控 (Motion JPEG 串流)
*   **低开销截图**: 客户端在配置开启屏幕捕捉后，由 xgb 异步捕获 X11 的 root window 的原始像素，并在内存中采用高性能算法将其快速压制为标准 JPEG 图像。
*   **分布式串流**: 客户端提供 `http://<client_ip>:8090/screen` 接口用于接收抓图串流。服务端控制台的“选手屏幕”监视器通过前端 JS 定时读取该客户端接口的单张图片（~4 FPS），从而将截图的网络带宽负荷全部分摊到管理员的浏览器终端，极大地降低了服务端的负载和网络拥堵可能性。

### 9. 批量文件分发管理 (Distribution Task)
*   **文件上传**: 允许管理员通过后台上传比赛题目 PDF、测试数据、大压缩包或环境补丁至服务端特定的暂存路径。
*   **批量推送流程**:
    1.  选择分发的设备子集，配置传输目标机目录（如 `/home/cwxu/Downloads`）。
    2.  支持下发前网络预检查 (Precheck) 控制，测试局域网网络可达性与访问凭证。
    3.  使用断点续传/分片技术传输，客户端拉取服务端文件。
    4.  实时向服务端上报传输的字节进度、百分比以及当前瞬时速率。
    5.  传输结束后自动进行 MD5 完整性校验，并可在客户端执行后置处理指令 (例如自动解压 `.tar.gz` 压缩包、分发题目后自动锁定浏览器等)。

### 10. 管理后台 Web 控制台模块

后台使用 jQuery 构建为单页面 Web 应用 (SPA) 风格的看板程序，拥有丰富的视觉反馈和实时状态订阅 (WebSocket API)：
1.  **控制台首页 (Dashboard)**: 动态图表反映全场机器上线数、在线百分比、签到人数、未签到人次、当前系统负荷及进行中的命令统计。
2.  **设备管理 (Devices)**:
    *   表格化展示所有录入设备的硬软件规格、网络端口与状态。
    *   支持单机信息深度抽屉页面板展示。
    *   提供批量强制重置(重分配ID)、删除死机节点、导出包含所有选手机配置的规范 `.xlsx` 报表功能。
3.  **批量命令 (Commands)**:
    *   左侧预设命令组选择器。
    *   中间集成 CodeMirror 编辑器，支持 Shell 脚本语法高亮和批量目标选手机筛选。
    *   下方控制台实时输出命令终端输出（按行流式渲染 stdout/stderr），并支持一键下发强杀中止信号。
4.  **屏幕监控 (Screen)**: 动态监控大厅，可按 ID 排列所有选手机的桌面缩略图。允许点击任何一台机器在模态弹窗中放大观看 1080P 高清的桌面对战画面。
5.  **文件分发 (Distribute)**: 文件存储管理平台，可管理上传包，指定下发路径，并发下发大包，并有完整的波形图或进度条反馈上传速控。
6.  **大屏广播与锁屏控制 (Broadcast)**:
    *   支持系统范围的消息广播（包括赛前、比赛中、赛后等各类静态或滚屏通告）。
    *   提供锁屏 (Lock/Unlock) 控制指令，支持一键锁定参赛者的桌面与浏览器，在赛前准备或赛后收卷时发挥关键的强制纪律功能。
7.  **系统配置 (Settings)**:
    *   管理签到文本信息配置 (欢迎信息、注意事项等)。
    *   设置屏幕截图捕捉使能状态。
    *   管理客户端的全局 ID 递增前缀和预设命令列表。
8.  **后台认证 (Auth)**: 使用基础凭证认证，保护大屏控制面板不被选手或非授权者非法访问。
